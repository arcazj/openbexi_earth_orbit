## Open Gates and Limitations

- A fresh loopback run rendered the visible globe in 0.189 seconds, but this single result is not percentile evidence and the 4-second catalog-dependent interactive target remains unmet. The performance release gate remains open.
- The profile is a single SwiftShader run. Repeated percentiles, hardware-GPU resource counts, and a 2x-catalog run are not available.
- The bundled data are stale for the screening-time policy, and 1,252 propagation failures make the observed result partial. Absence of an event is not proof of safety.
- Collision probability remains unavailable because covariance, covariance-frame support, correlated uncertainty treatment, and hard-body radii are not provided.
- Full-catalog scheduled jobs, CDM/OMM/OEM ingestion, persistent event history/import, alerts, maneuver recommendations, and operational decision support are not implemented in v2.0.
- Validation uses deterministic reference and synthetic cases but has not received independent scientific review.
- Broader legacy rendering code has not fully migrated to the new propagation service.
- The tracked generated Java class under `target/` remains cleanup debt; it was not removed because existing workspace files were preserved.
- Asset redistribution/licensing review, independent security review, clean external CI, public hosting/TLS/monitoring, and rollback rehearsal remain incomplete.

No v2.1 work is approved by this evidence.

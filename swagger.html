OpenBEXI Earth Orbit API 2.1.0 OAS 3.0.3

Standard Swagger/OpenAPI-style local reference for the optional Python API used by OpenBEXI Earth Orbit. This page is static and self-contained, so it can be displayed without starting server.py. Live requests, Try-it-out behavior, /docs, and /openapi.json require the optional local server.

Base URL / Schema Source Static page: swagger.html Optional live API: http://127.0.0.1:8000 Optional live schema: http://127.0.0.1:8000/openapi.json
No Python Required

Use this page and the companion Markdown reference from local static files. The optional API server is only needed for live JSON responses.

Method Colors

GET is blue, POST is green, PUT is orange, and DELETE is red.

Local Assets

The styling and behavior are implemented inline. There is no CDN or remote Swagger UI dependency for displaying this page.

screening v1 Authenticated durable full-catalog jobs and immutable event history

GET /api/v1/capabilities Discover Experimental server capability, bounds, and availability.

Public discovery response. Liveness is available at /api/v1/health/live; readiness is available at /api/v1/health/ready.

Responses

200Capability document.application/json
GET /api/v1/catalog-revisions List immutable catalog revisions with signed keyset cursors.

Requires a viewer, analyst, or administrator bearer token. Supports limit, cursor, source_id, and source_status.

Related

GET /api/v1/catalog-revisions/{revision_id} reads one revision; use current for the active revision.

POST /api/v1/screening-jobs Submit an idempotent bounded full-catalog screening job.

Requires an analyst or administrator bearer token plus an Idempotency-Key. The request freezes catalog revision, scope, start time, numerical configuration, resource caps, timeout, and attempt budget.

Responses

202Queued or idempotently replayed job.application/json
400Invalid or unbounded request.application/problem+json
401 / 403Missing credential or insufficient role.application/problem+json
409Idempotency key reused for different content.application/problem+json
GET /api/v1/screening-jobs/{job_id} Read state, attempts, progress, and the scientific result summary.

GET /api/v1/screening-jobs lists jobs. DELETE on the detail URL requests cancellation. POST .../retry retries an eligible attempt and POST .../replay creates a new job from the frozen inputs.

GET /api/v1/screening-jobs/{job_id}/stream Resume authenticated progress using Server-Sent Events.

Requires an Authorization header. Resume with Last-Event-ID; tokens in URLs are rejected. Browser clients use streaming fetch, not native EventSource.

Response

200 text/event-stream with durable job.state, job.progress, and job.result records.

GET /api/v1/conjunction-events Query filtered immutable event revisions.

Supports signed cursor pagination, job/object filters, TCA bounds, maximum miss distance, and stable TCA or miss-distance ordering. Read one revision at /api/v1/conjunction-events/{event_id}.

Results remain Experimental geometric screening. Collision probability is unavailable.

Authentication

Configure role tokens with OPENBEXI_API_VIEWER_TOKEN, OPENBEXI_API_ANALYST_TOKEN, and OPENBEXI_API_ADMIN_TOKEN. Send tokens only in the Authorization: Bearer header.

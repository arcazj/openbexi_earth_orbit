OpenBEXI Earth Orbit API 1.7.4 OAS 3.0.3

Standard Swagger/OpenAPI-style local reference for the optional Python API used by OpenBEXI Earth Orbit. This page is static and self-contained, so it can be displayed without starting server.py. Live requests, Try-it-out behavior, /docs, and /openapi.json require the optional local server.

Base URL / Schema Source Static page: swagger.html Optional live API: http://127.0.0.1:8000 Optional live schema: http://127.0.0.1:8000/openapi.json
No Python Required

Use this page and the companion Markdown reference from local static files. The optional API server is only needed for live JSON responses.

Method Colors

GET is blue, POST is green, PUT is orange, and DELETE is red.

Local Assets

The styling and behavior are implemented inline. There is no CDN or remote Swagger UI dependency for displaying this page.

system Health, version, and local server metadata

GET /api/health Check server health and release metadata.

Description

Returns a small JSON payload that confirms the optional local Python API is available.

Parameters

No parameters.

Responses

StatusDescriptionContent type
200Server is available.application/json

Example Response

{
  "status": "ok",
  "app": "OpenBEXI Earth Orbit",
  "version": "1.7.4",
  "release_date": "2026-06-14"
}

Schema

{
  "type": "object",
  "additionalProperties": true
}

satellites TLE, satellite records, metadata, and decay data

GET /api/tle Load the TLE dataset used by the frontend.

Description

Returns the active satellite TLE array from json/tle/TLE.json. The frontend falls back to local files if this endpoint is unavailable.

Parameters

No parameters.

Responses

StatusDescriptionContent type
200TLE satellite records.application/json

Example Response

[
  {
    "name": "ISS (ZARYA)",
    "noradId": 25544,
    "line1": "1 25544U ...",
    "line2": "2 25544 ..."
  }
]

Schema

{
  "type": "array",
  "items": {
    "type": "object",
    "additionalProperties": true
  }
}

GET /api/satellite-metadata List available satellite metadata files.

Description

Lists safe JSON metadata files available under json/satellites/.

Parameters

No parameters.

Responses

StatusDescriptionContent type
200Metadata file index.application/json

Example Response

{
  "files": [
    {
      "name": "starlink_V1.json",
      "url": "/api/satellite-metadata/starlink_V1.json"
    }
  ]
}
GET /api/satellite-metadata/{file_name} Load one known satellite metadata JSON file.

Description

Returns a single metadata JSON file by safe file name. Unknown or unsafe names return an error.

Parameters

NameInRequiredSchemaDescription
file_namepathYesstringMetadata file name, for example starlink_V1.json.

Responses

StatusDescriptionContent type
200Satellite metadata JSON.application/json
404Metadata file not found.application/json

Example Response

{
  "name": "Starlink V1",
  "model": "obj/starlink_V1.obj",
  "texture": "obj/Textures/starlink_BaseColor.png"
}
GET /api/display-satellite-models List GLB and OBJ/MTL satellite models under obj.

Description

Returns a live display-model manifest by scanning obj/ recursively for every .glb file and every .obj file with a matching .mtl.

Parameters

No parameters.

Responses

StatusDescriptionContent type
200Display satellite model manifest.application/json

Example Response

{
  "schemaVersion": 1,
  "basePath": "obj/",
  "models": [
    {
      "id": "starlink_V1",
      "type": "obj-mtl",
      "files": {
        "obj": "starlink_V1.obj",
        "mtl": "starlink_V1.mtl"
      }
    },
    {
      "id": "ISS.glb",
      "type": "glb",
      "files": {
        "glb": "ISS.glb"
      }
    }
  ]
}

GET /api/data-update-status Read optional scheduled data-update status.

Description

Returns the current background data-update scheduler state. The scheduler is disabled by default and only runs when server.py starts with --update-data-on-schedule.

Parameters

No parameters.

Responses

StatusDescriptionContent type
200Scheduler status snapshot.application/json

Example Response

{
  "enabled": false,
  "state": "disabled",
  "interval_hours": 24
}
